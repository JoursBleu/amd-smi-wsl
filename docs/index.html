amd-smi-wsl

A drop-in replacement for the amdsmi Python package that works inside WSL2 / Windows — where the native AMD SMI library can't run — so vLLM and other ROCm tools detect your GPU and start cleanly. Works across RDNA-generation Radeon / Radeon PRO GPUs and Ryzen AI APUs on WSL2.

View on GitHub View on PyPI
pip install amd-smi-wsl

Why it exists

On WSL2 the GPU is exposed through DirectX para-virtualisation — not the native KFD driver.

Under WSL2 any AMD GPU is reached through /dev/dxg + dxgkrnl, not the native amdgpu KFD driver. The Linux /dev/kfd device and its sysfs topology simply do not exist, so import amdsmi fails and downstream tools crash during ROCm platform detection — even though the HIP runtime itself works perfectly. Because this is a property of the WSL2 stack, it affects Radeon / Radeon PRO GPUs and Ryzen AI APUs alike, across RDNA generations.

native amdsmi import ✗ → ROCm platform detection ✗ → vLLM startup crash 💥
install amd-smi-wsl ✓ → queries answered via HIP + Windows interop ✓ → vLLM runs 🚀

What it does

Binary-compatible import surface, real answers where the platform allows.

🔌

100% import-compatible

Exposes every public symbol of the upstream binding — all 190 amdsmi_* functions, all enums and the full exception hierarchy. import amdsmi just works.

🧠

Real data sources

Device name, GCN arch, VRAM, compute units, UUID and live memory come from torch.cuda (HIP); PCI id, subsystem id, revision and driver info come from Windows interop.

♻️

torch re-entrancy safe

PyTorch's ROCm build counts devices through amdsmi. A thread-local guard breaks the recursion so torch falls back to its native HIP device count.

🎯

vLLM canonical names

Returns device_id as a hex string ("0x1586") so vLLM resolves the canonical name from its device map, e.g. AMD_Radeon_8060S.

🧪

Honest stubs

Features the platform genuinely lacks (RAS/ECC, partitioning, set-mutators, events…) raise AMDSMI_STATUS_NOT_SUPPORTED — exactly like the native library.

📦

Zero hard deps

Pure-Python, no compiled extensions. torch is expected to be present already and is intentionally not pinned, so it never fights your ROCm install.

Quickstart

Install where the real amdsmi can't run, then use it like the original.

import amdsmi            # this package, not the native one

amdsmi.amdsmi_init()
h = amdsmi.amdsmi_get_processor_handles()[0]
info = amdsmi.amdsmi_get_gpu_asic_info(h)
print(info["market_name"])   # AMD Radeon(TM) 8060S Graphics
print(info["device_id"])     # 0x1586

Set AMDSMI_WSL_DISABLE=1 to make amdsmi_init() raise NOT_SUPPORTED for testing a caller's fallback path.

Verified end-to-end

GPU-agnostic by design — validated on WSL2 + AMD Radeon 8060S (Strix Halo, gfx1151) + ROCm 7.2.4 + PyTorch 2.9.1; device-specific values differ on other cards.
